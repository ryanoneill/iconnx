Steps use checkbox (`- [ ]`) syntax for tracking. Each milestone is one (occasionally split) GPG-signed feat commit. Escalation discipline: if garboard / ort / cuDNN signatures or behaviors surprise you (an upstream-contract mismatch), STOP and escalate the task as BLOCKED — never work around an upstream contract bug inside a test (per `feedback_subagent_escalation`).

**Goal:** Make iconnx's CUDA `GpuGraphExecutor` run both pinned Apache-2.0 OCR models (`en_number_mobile_v2.0_rec` and `en_PP-OCRv3_det`) end-to-end with ONNX-Runtime-validated correctness, consumable by pdfocr through a one-call `from_model` API plus a pre-flight support check. This workstream corrects the op-diff's central misconception: op-NAME coverage ≠ semantic coverage. Two model-critical gaps hide behind "supported" ops (grouped Conv2D panics; input-form Clip silently no-ops), plus four genuinely-missing ops (BatchNormalization, HardSigmoid, HardSwish, Tile).

**Architecture:** Six dependency-ordered milestones (Approach A — model-independent work first to unblock pdfocr's API wiring earliest). M6.1 API + live-ORT test harness + carry-forward doc hygiene. M6.2 R2 dynamic-shape correctness + always-on shape guard. M6.3 conv correctness (grouped Conv2D via cuDNN + input-form Clip + ConvTranspose-2D parity). M6.4 four missing ops. M6.5 whole-model R3/R6 parity (blocked on pdfocr fixtures). M6.6 R5 characterization + batching/perf follow-up doc. fp32 only; batch=1; one most-recent plan. M6.1–M6.4 are unblocked; **M6.5 is BLOCKED on pdfocr delivering two preprocessed `.npy` input fixtures + their SHA-256s and the two `_rapidai` model exports**.

**Tech Stack:** Rust 1.87 (matches `rust-version`), `garboard` (sole CUDA dep: device/stream/memory/NVRTC/cuBLAS/cuDNN — to be pinned to `rev="75a1f66071a700c97e60070c39d89d6d04e15e90"`), `ort = "2.0.0-rc.10"` (dev-dep, features `cuda,half,download-binaries` — **first live `ort::` use lands in M6.1**), `npyz = "0.8"` (dev-dep — `.npy` fixture loader for M6.5), `tempfile`, `regex` (drift fence), `toml` (conformance allowlist), `half`, `prost`. No new runtime deps; M6.5 adds a `sha2` dev-dep for fixture SHA-256 verification (O-6 resolved). `GpuGraphExecutor` is `!Sync` (interior `RefCell`s); the live-ORT helper and parity tests are single-threaded.

**Decision log (from spec §3 — baked in, no re-litigation):**
- R2 (§3.1): error guard ALWAYS-ON (MUST) + verify dynamic-load correctness (MUST); concrete-load re-plan is a SHOULD (the guard is the floor that prevents silent miscompute if deferred). Re-lowering re-uploads initializers (cheap, sub-ms) but does NOT recompile NVRTC kernels (cached on `Executor`). Keep ONE most-recent plan (multi-entry shape-keyed cache deferred). The guard runs before the baked plan is used → baked Reshape/Slice params are covered by construction (no separate baked-param check).
- R3/R6 (§3.2): whole-model parity gate uses REALISTIC preprocessed input (random `N(0,1)` is degenerate — rec → all-blank CTC, det → all-zero mask). Tensor gate `atol=1e-3, rtol=1e-3`. Semantic gates: rec CTC-greedy decoded sequence exact-match + **non-degeneracy guard** (ORT reference must have ≥1 non-blank timestep); det prob-map closeness + binary-mask IoU@0.3 + **non-empty-mask guard**. Per-op tests (M6.3/M6.4) keep seeded-random fixtures (fine in isolation). R6 Resize parity folds into the det test (all 6 DET Resize are `nearest/asymmetric/floor`).
- R5 (§3.3): batching deferred — characterize only (latency + GPU-mem + confirm batch=1) and write a follow-up doc.
- Conv routing (§3.5): route `group>1` (and any dilated) Conv2D through garboard cuDNN; leave `group==1` on the validated im2col+GEMM `gpu_conv2d` path.
- Oracle for M6.2: the M6.1 live-ORT helper — **NOT** the CPU `GraphExecutor` (its 2D Conv hardcodes stride=1/pad=0, `src/operators/conv.rs:~222`).

**Verified caveats carried from spec §5 (note in code/docs, out of scope to fix):** `Softmax` ignores the `axis` attribute and always reduces the last axis (`elementwise.rs:285`); REC's `axis=2` on rank-3 = the last axis so it is correct *for these models only*. `LSTM` ignores `sequence_lens` (harmless for batch=1 full-length OCR).

  /// Build a `GpuGraphExecutor` directly from a parsed `OnnxModel`.
  ///
  /// Seeds inputs from `model.input_shapes()`, uploads every initializer via
  /// `add_initializer`, and adds each `computation_graph()` node via `add_node`.
  /// `GraphNode` carries no `name` field, so node names are synthesized as
  /// `n{i}` over the graph's topological node order (the names are internal —
  /// only input/output tensor names participate in dataflow).
  ///
  /// This does NOT call `validate_*`; callers wanting a fail-fast op-name
  /// pre-flight should call `validate::validate_parsed_model(&model)` first
  /// (op-NAME presence only — not a runnability guarantee).
  pub fn from_model(model: &crate::onnx_parser::OnnxModel) -> crate::Result<Self> {
      let mut exec = Self::new()?;
      for (name, shape) in model.input_shapes() {
          exec.add_input(name, shape);
      }
      let weights = model.extract_weights()?;
      for (name, tensor) in weights {
          exec.add_initializer(name, &tensor)?;

      let graph = model.computation_graph()?;
      for (i, node) in graph.nodes().iter().enumerate() {
          let synth = format!("n{i}");
          exec.add_node(
              &synth,
              &node.op_type,
              node.inputs.iter().map(|s| s.as_str()).collect(),
              node.outputs.iter().map(|s| s.as_str()).collect(),
              node.attributes.clone(),
          );
      }
      Ok(exec)
  }

  ```rust
  pub fn garboard_conv_2d(
      ctx: &IconnxCudaContext,
      input: &GpuTensor,   // [N, C, H, W]
      kernel: &GpuTensor,  // [Cout, Cin/groups, kH, kW]
      params: &crate::cuda::conv::Conv2dParams,
  ) -> Result<GpuTensor, CudaError> {
      let s = input.shape();
      if s.len() != 4 { return Err(CudaError::Cudnn(format!("Expected 4D input [N,C,H,W], got {s:?}"))); }
      let k = kernel.shape();
      if k.len() != 4 { return Err(CudaError::Cudnn(format!("Expected 4D kernel, got {k:?}"))); }
      let (n, c, h, w) = (s[0], s[1], s[2], s[3]);
      let (cout, cin_g, kh, kw) = (k[0], k[1], k[2], k[3]);
      let (ph, pw) = (params.pad_h, params.pad_w);
      let (sh, sw) = (params.stride_h, params.stride_w);
      let (dh, dw) = (params.dilation_h, params.dilation_w);
      let out_h = (h + 2*ph - dh*(kh-1) - 1)/sh + 1;
      let out_w = (w + 2*pw - dw*(kw-1) - 1)/sw + 1;
      let config = garboard::ConvForwardConfig {
          input_dims: [n as i32, c as i32, h as i32, w as i32],
          filter_dims: [cout as i32, cin_g as i32, kh as i32, kw as i32],
          padding: [ph as i32, pw as i32],
          stride:  [sh as i32, sw as i32],
          dilation:[dh as i32, dw as i32],
          groups: params.group as i32,
          algorithm: None,
      };
      let mut output = GpuTensor::zeros_f32(ctx, vec![n, cout, out_h, out_w])?;
      ctx.dnn().conv_forward(ctx.garboard_stream(), &config, input.data_f32()?, kernel.data_f32()?, output.data_f32_mut()?)
          .map_err(|e| CudaError::Cudnn(e.to_string()))?;
      Ok(output)
  }
  ```

  // ndim<=8; in_shape/out_shape passed as device or kernel params.
  extern "C" __global__ void tile_kernel(
      float* out, const float* x, size_t n_out, int ndim,
      const long long* in_shape, const long long* out_strides, const long long* in_strides
  ) {
      size_t o = blockIdx.x * blockDim.x + threadIdx.x;
      if (o >= n_out) return;
      long long rem = (long long)o, in_idx = 0;
      for (int d = 0; d < ndim; ++d) {

      out[o] = x[in_idx];
  }
